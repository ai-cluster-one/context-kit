# Runtime Standard

Rule source for ContextKit's always-on operating runtime block.

## Contract

Runtime context directs an operating agent already inside a ContextKit-managed
project. It gives orientation, law, routing, safety, source-edit discipline, and
compact awareness of on-demand surfaces.

Runtime does not document the whole product, repeat every guide, copy command
contracts, narrate migration history, or carry rare authoring detail.

## Content Bar

Include only material that most sessions need to stay safe, oriented, and
aligned:

- the source body versus generated output distinction;
- body-layer routing;
- command-discovery and rule-source routing;
- repair source discipline;
- validation loop.

## Quality Bar

Runtime is good when it lets the agent operate immediately while making the next
source owner or guide obvious.
