# Altitude

Power: placement.

Every piece of knowledge belongs at the lowest durable altitude that can own it
without losing meaning.

Decision Question: where should this knowledge live so owner, scope, and
consumption match its truth?

## Law

Put a fact where it is true, owned, and useful:

- canon owns cross-surface laws;
- standards apply canon to authored surfaces;
- bundle ships runtime blocks, guides, templates, and product routines;
- project context owns current project doctrine;
- assets preserve evidence and history;
- routines own repeatable procedure;
- tools own command-answerable contracts;
- generated files deliver compiled views.

Choose altitude before copying. Once altitude is chosen, One Fact, One Home
keeps the owner singular.

## Failure Signs

- product canon contains one project's local decision;
- runtime explains how to build ContextKit instead of how to operate;
- a guide repeats executable command contracts;
- always-on text carries rare authoring detail.
