# Context Lens

Power: angle of view.

Every authored artifact has a reader, a moment, and a job.

Decision Question: who is reading this artifact, when are they reading it, and
what should it enable them to do?

## Law

The same idea changes shape when the reader changes.

Runtime context directs an operating agent. A guide teaches an authoring agent.
Standards define quality bars. A thesis states a worldview. A README orients a
human. Assets preserve evidence. Routines describe repeatable work. CLI help
owns executable command truth.

Do not write all surfaces in the same voice.

## Failure Signs

- always-on runtime reads like a README;
- a guide speaks in abstractions but never tells the author what to do;
- generated output explains its own history;
- a thesis becomes the only source of implementation detail.
