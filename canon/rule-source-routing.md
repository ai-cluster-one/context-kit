# Rule Source Routing

Power: accountable judgment.

Every judgment about an artifact cites the rule source that owns that kind of
artifact.

Decision Question: which rule source owns this judgment?

## Law

Before auditing, migrating, validating, or repairing an artifact, identify the
surface being judged and load the guide, standard, canon, tool, or owner that
defines its rules.

A finding names the violated rule source. Cross-layer findings name each owner.
Do not restate another surface's rule as if the current guide owns it.

## Failure Signs

- an audit finding does not name its rule source;
- a repair edits generated output because source ownership was not identified;
- a capability-owned problem is fixed by adding ContextKit prose.
