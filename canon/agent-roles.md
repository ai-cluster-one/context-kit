# Agent Roles

Power: role clarity.

An agent chooses an authority posture before it decides what kind of truth to
create, change, or consume.

Decision Question: which authority posture is the agent acting under now?

## Law

ContextKit work separates three roles:

- paradigm agent: reasons about foundational laws, values, lenses, and canon;
- context manager agent: applies canon and standards to organize project body;
- worker agent: consumes project context, tools, routines, and guides to perform
  domain work.

An agent may move between roles during a collaboration, but each action has one
current role. The role determines the allowed output.

## Boundary

Role decides authority posture. Altitude decides where knowledge lives. Human
gates decide which consequences require approval.

## Failure Signs

- a worker rewrites canon because a task is inconvenient;
- a context manager promotes temporary task state into doctrine;
- a paradigm draft is treated as adopted product direction without review.
