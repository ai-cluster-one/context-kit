# Present-Tense Doctrine

Power: live clarity.

Live doctrine states the current form of the project.

Decision Question: is this current doctrine, or a trace of how the doctrine was
reached?

## Law

When a conclusion becomes live doctrine, state it as the final form the agent
should act on now.

Live doctrine states what is true, what to do, and which boundaries matter. It
does not include internal arguments, discarded alternatives, migration history,
or self-justifying prose.

## Failure Signs

- doctrine defines itself by what it replaced;
- a migration note remains in runtime context;
- a file makes the agent infer whether a paragraph is current law or history.
